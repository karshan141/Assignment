02.   How is the use of comments and how do you write comments in HTML?

In programming, comments are lines of text that are added to the code to provide explanations, documentation, or reminders to the developer who reads the code in the future. Comments are ignored by the computer when the code is executed, and they do not affect the output or behavior of the program.

In HTML, comments are created using the <!-- and --> tags. Anything written between these tags will be considered as a comment and will not be displayed in the output.

03.   What are elements, tags, and attributs and the differnce between them?

  1. Elements: An HTML element is a structural component of a webpage, such as a header, paragraph, or image. Each element is identified by an opening tag, which defines the type of element, and a closing tag, which indicates the end of the element's content. For example, the following HTML code creates a paragraph element:
    <p> This is a paragraph element. <p>
  2. Tags: HTML tags are used to define the structure and content of an element. Tags are enclosed in angle brackets (<>) and come in pairs: an opening tag and a closing tag. The opening tag includes the name of the element, and the closing tag includes a forward slash (/) before the element name. Some elements, such as images and line breaks, do not require a closing tag. For example, the following HTML code creates a bold element:
    <b>This text is bold.</b>
  3. Attributes: HTML attributes provide additional information about an element, such as its ID, class, or source URL. Attributes are included within the opening tag and are written in the format name="value". For example, the following HTML code creates an image element with a specified source URL and alt text:
    <img src="image.jpg" alt="A beautiful sunset">

In summary, elements are the building blocks of HTML documents, tags are used to define the structure and content of elements, and attributes provide additional information about elements.

04.   Briefly explain what HTML entities are.

HTML entities are special characters that are used to represent reserved characters or characters that are not part of the standard character set in HTML code. They are represented by a string of characters that starts with an ampersand (&) and ends with a semicolon (;).

For example, the less-than sign (<) is a reserved character in HTML code, and if you want to use it as a regular character, you need to represent it using the HTML entity &lt; Similarly, if you want to use non-English characters, like accented letters or special symbols, in your HTML code, you need to use the corresponding HTML entity to represent them.

Using HTML entities helps ensure that your code is valid and that your content is displayed correctly in all browsers and devices, regardless of their character encoding settings.
